# 技術仕様書 (Architecture Design Document)

## テクノロジースタック

### 言語・ランタイム

| 技術 | バージョン | 備考 |
|------|-----------|------|
| HTML | 4.01 Transitional | 2003年政府サイト再現のためHTML5非推奨タグも許容 |
| CSS | 2.1 相当 | flexbox/grid 不使用、float レイアウト |
| JavaScript | ES2020+ (Vanilla) | フレームワーク・ビルドツール不使用 |

### フレームワーク・ライブラリ

| 技術 | バージョン | 用途 | 選定理由 |
|------|-----------|------|----------|
| Chart.js | 4.x (CDN) | 嘘残量折れ線グラフ | 軽量・設定容易・外部依存1本のみ |
| @vladmandic/face-api | 1.7.15 (ローカル) | 嘘判定ページの顔検出 HUD | tinyFaceDetector のみ、TF.js 同梱、診断結果はランダム |

### 開発ツール

| 技術 | バージョン | 用途 | 選定理由 |
|------|-----------|------|----------|
| VS Code / 任意エディタ | — | コード編集 | 静的ファイルのみのため制約なし |
| ブラウザ（Chrome推奨） | 最新 | デモ実行・動作確認 | Webカメラ API の対応が安定 |

**ビルドステップ不要**: npm install / webpack / Vite 等は使用しない。ファイルをブラウザで直接開くか、`python -m http.server` 等の簡易HTTPサーバーで確認する。

## アーキテクチャパターン

### シングルページ静的ウェブアプリ（マルチHTML）

```
┌────────────────────────────────────────┐
│  プレゼンテーション層                    │
│  HTML + CSS + Vanilla JS              │
│  （ビジネスロジック・データを内包）       │
└────────────────────────────────────────┘
           ↑
     すべてブラウザ内で完結
     外部API / サーバー通信: なし
```

バックエンドを持たない完全静的構成。データはすべてJavaScript定数としてハードコードし、インターネット非接続環境でも動作する。

### ページ構成

| ファイル | 役割 |
|---------|------|
| `src/index.html` | ダッシュボード（メイン画面） |
| `src/detection.html` | 嘘判定ページ |
| `src/savings.html` | 嘘節約のすすめページ |
| `src/css/style.css` | 全ページ共通スタイル |
| `src/js/data.js` | 嘘残量・事件・判定パターンのデータ定数 |
| `src/js/dashboard.js` | 嘘残量グラフ描画ロジック |
| `src/js/detection.js` | Webカメラ・face-api 顔検出 HUD・嘘判定演出ロジック |
| `src/js/vendor/face-api.min.js` | face-api.js + TensorFlow.js 同梱（嘘判定ページのみ） |
| `src/models/tiny-face-detector/` | tinyFaceDetector モデル weights |

## データ永続化戦略

| データ種別 | ストレージ | フォーマット | 理由 |
|-----------|----------|-------------|------|

## パフォーマンス要件

### レスポンスタイム

| 操作 | 目標時間 | 備考 |
|------|---------|------|
| ページ初期表示 | 2秒以内 | Chart.js CDN読み込み含む |
| Webカメラ起動 | 3秒以内 | ブラウザ権限ダイアログ含まず |
| 嘘判定結果表示 | 演出3〜5秒後 | 意図的な遅延（演出） |
| ページ間遷移 | 1秒以内 | 静的HTML遷移 |

### リソース使用量

| リソース | 上限 | 理由 |
|---------|------|------|
| 合計ファイルサイズ | 5MB以下 | Chart.js CDN + TF.js/face-api ローカル vendor + モデル |
| メモリ（ブラウザ） | 100MB以下 | Webカメラストリームを含む |

## セキュリティアーキテクチャ

### データ保護

- **Webカメラ映像**: `getUserMedia()` で取得したストリームは `<video>` 表示と face-api による顔枠検出（HUD 演出）にのみ使用。静止画キャプチャ・外部送信は行わない
- **個人情報**: 収集・送信するデータなし
- **外部通信**: ダッシュボードの Chart.js CDN 読み込み1件のみ（嘘判定ページはローカル vendor でオフライン動作可能）

### 入力検証

- ユーザー入力フォームなし（クリック操作のみ）
- DOM操作は `textContent` を使用。`innerHTML` は静的テンプレートの挿入のみとし、ユーザー入力値を含めない

## 技術的制約

### 環境要件

| 要件 | 内容 |
|------|------|
| OS | 不問（ブラウザが動作する環境） |
| ブラウザ | Chrome / Edge 最新版（Webカメラデモ用）|
| インターネット | Chart.js CDN を使用する場合は必要。オフライン化する場合はローカルコピー |
| Webカメラ | 嘘判定機能に必要（非搭載でも結果は表示） |

### 意図的な技術制約（デザイン要件）

| 制約 | 理由 |
|------|------|
| CSS flexbox/grid 不使用 | 2003年政府サイト再現（float レイアウト） |
| `font-family` に MS Pゴシック 指定 | 年代考証 |
| 800px 固定幅レイアウト | 2003年当時の標準ディスプレイ幅 |
| レスポンシブ対応なし | スマートフォン対応はスコープ外 |
| モダンCSS変数・カスタムプロパティ 最小使用 | 旧時代感の維持 |

## スケーラビリティ設計

ハッカソン用のデモアプリケーションであり、スケーラビリティは設計要件外。

### 機能拡張性

- グラフデータの追加: `src/js/data.js` の定数配列を編集するだけ
- 嘘判定パターンの追加: `LIE_JUDGMENT_PATTERNS` 配列に要素を追加するだけ
- 新規ページの追加: 同じ共通レイアウトHTMLを複製して編集

## テスト戦略

ハッカソン用プロジェクトのため、自動テストは不要。以下の手動テストのみ実施する。

### 動作確認チェックリスト

- [ ] Chrome（最新版）でダッシュボードが表示される
- [ ] グラフが正しく描画される（折れ線、右肩下がり、マーカー表示）
- [ ] TOP5テーブルが表示される
- [ ] 解析中に顔枠 HUD が表示される（カメラ・モデルロード成功時）
- [ ] 3〜5秒後に結果が表示される（毎回異なること）
- [ ] 「再解析」ボタンで新しい結果が表示される
- [ ] 節約のすすめページが表示される
- [ ] 左サイドメニューのダミーリンクがクリックできない（またはジャンプしない）
- [ ] 800px固定幅で表示が崩れない
- [ ] プロジェクターで投影した際（1024×768）に視認できる文字サイズ

## 依存関係管理

| ライブラリ | 用途 | バージョン管理方針 |
|-----------|------|-------------------|
| Chart.js | 嘘残量グラフ | CDNバージョン固定（`@4.4.0`相当） |
| @vladmandic/face-api | 顔検出 HUD（TF.js 同梱） | ローカル固定（`src/js/vendor/face-api.min.js` `@1.7.15`） |
| tinyFaceDetector | 顔検出モデル | ローカル固定（`src/models/tiny-face-detector/`） |

**CDNリンク例**:
```html
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js"></script>
```

オフライン対応が必要な場合は上記ファイルをダウンロードして `src/js/vendor/chart.umd.min.js` に配置する。
