# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**嘘資源管理庁** — Ministry of Lie Resources  
サブタイトル：「嘘の枯渇から、未来を守る。」

### プロダクトコンセプト
- **嘘は有限資源である**: 石油・電力と同じ構造で「嘘の残量」を管理する架空の政府機関ポータルサイト
- **サイト自体が嘘というメタ構造**: プロダクト全体が「嘘」というテーマへの回答になっており、種明かしで電力問題という真実を伝える
- **平成レトロ政府サイト**: 2003〜2007年ごろの日本の政府系Webサイトを忠実に再現した意図的な「古さ」でリアリティと笑いを両立する

### プロダクトビジョン
世界の嘘の総量は決まっていた。AIの台頭により嘘（ハルシネーション）は急速に消費されており、このままでは人類はあと数年で嘘がつけなくなる。
架空の政府機関「嘘資源管理庁」が運営するポータルサイトとして、嘘の資源枯渇を真顔で訴える。
3分のハッカソンプレゼンを通じて審査員を完全に騙し、最後の15秒の種明かし「これは嘘ですが、まったく同じことが電力の世界で起きています」で笑いと問題提起を同時に届ける。

### 目的
- ハッカソンのテーマ「嘘」に対して、プロダクト自体が嘘という最もメタなアンサーを提示する
- 種明かしを通じて、AI の電力消費問題という本物の課題に聴衆の関心を向ける
- 「アホなことに本気で取り組む」姿勢で、審査員・観客の記憶に残るプレゼンを実現する

---

## ターゲットユーザー

### プレゼン中のターゲット：審査員（ハッカソン当日）

**ペルソナ: 藤井亮（映像作家・クリエイティブディレクター）**
- クリエイティブの文脈を熟知しており、「変なモノ」への感度が高い
- 整いすぎたプロダクトより、ディテールへの偏執と余白のあるプロダクトを評価する
- 「2003年8月14日 14:22 急落（原因不明）」のような引用したくなる具体性を好む
- 種明かし後の「なるほど、だから本気でやってたのか」という反転を楽しめる

**課題**: 毎回似たようなプロダクトを見せられており、本当に笑えて、かつメッセージのあるものを求めている  
**期待する解決策**: 徹底的に真顔で押し通す演技と、精巧に作られた「嘘っぽくない嘘サイト」

### 公開後のターゲット：SNSユーザー（種明かし後）

**ペルソナ: 一般的なTwitter/Xユーザー（25〜40代）**
- 「面白いが、実は意味もある」コンテンツをシェアしたい
- 「2003年8月14日 14:22 急落（原因不明）」のような細部に反応する
- 日本の古い政府サイトへのノスタルジーがある

## 成功指標（KPI）

### ハッカソン当日（プレゼン）
- 審査員が2:45まで「本物の政府系ポータル」と信じて疑わない（演者の主観的判断）
- 種明かし後に笑い・ざわめきが起きる（会場の反応）
- 審査員の1人以上が「2003年8月14日 14:22 急落（原因不明）」などの細部を指摘する

### SNS拡散（公開後1週間）
- プレゼン動画または画面キャプチャが Twitter/X で 1,000 インプレッション以上
- 「これは嘘ですが」の種明かし部分がクリップとして拡散される

### 制作品質
- 1024×768 解像度でも「2003年の政府サイト」に見える（審査員のレトロ感認知）
- Webカメラ判定の結果テキストが「やたら具体的」であると笑いを生む

## 機能要件

### コア機能（MVP）

#### 機能1: 嘘残量ダッシュボード（メイン画面）

**ユーザーストーリー**:  
プレゼンターとして、審査員に嘘が有限資源であると信じ込ませるために、公式統計のように見える嘘残量グラフを見せたい

**受け入れ条件**:
- [x] 折れ線グラフで「世界の嘘残量（単位：キロウソ / Kuso）」を右肩下がりで表示する
- [x] グラフ上に歴史的イベントマーカーを表示する（最低5点）
  - 2016年：米大統領選（急落）
  - 毎年4/1：エイプリルフール（急落）
  - 2003年8月14日 14:22：急落（原因不明）
  - 2022年：AI台頭以降（加速的急落）
  - その他1点以上（2008年リーマンショック関連説明）
- [x] グラフに「現在の残量」「消費速度（前年比）」「枯渇予測年」を数値表示する
- [x] 政府機関らしい「更新日時」「データ出典」の表示がある

**優先度**: P0（必須）  
**実装**: `src/index.html`, `src/js/data.js`, `src/js/dashboard.js`, `src/css/style.css`（2026-05-30 完了）

---

#### 機能2: 嘘大量消費事件 TOP5 テーブル

**ユーザーストーリー**:  
プレゼンターとして、嘘資源消費の深刻さを「統計」で見せるために、歴史的な嘘大量消費事件ランキングを表示したい

**受け入れ条件**:
- [x] テーブル形式で事件名・年・消費量（Kuso）・消費率を表示する
- [x] 5件以上のでたらめだが具体的な事件が記載されている
- [x] テーブルデザインが政府系の表組みスタイル（罫線、薄いグレー背景）

**優先度**: P0（必須）  
**実装**: `src/index.html`（ダッシュボードに統合、2026-05-30 完了）

---

#### 機能3: あなたの嘘判定（Webカメラ機能）

**ユーザーストーリー**:  
プレゼンターとして、審査員を直接参加させるために、Webカメラで「嘘タイプ診断」を実演したい

**受け入れ条件**:
- [ ] ボタンクリックでWebカメラを起動できる
- [ ] 「解析中...」アニメーション（3〜5秒）が表示される
- [ ] 結果が「やたら具体的でくだらない」文体で表示される
  - 例：「あなたがつきやすい嘘：**『一回家に取りに帰ります』系（73.2%）**」
  - 例：「直近の嘘リスク：**高（警戒レベル3）**」
- [ ] 結果は毎回異なる（複数パターンからランダム表示）
- [ ] 「再解析」ボタンがある
- [ ] 実際のカメラ画像は判定結果に影響しない（純粋に演出）

**優先度**: P0（必須）

---

#### 機能4: 嘘資源節約のすすめ（サブページ）

**ユーザーストーリー**:  
プレゼンターとして、「役所らしい啓発コンテンツ」を見せるために、嘘節約tips のページを用意したい

**受け入れ条件**:
- [ ] 5つ以上の「今日からできる嘘節約アクション」が記載されている
- [ ] 「① ささいな見栄を控えましょう」などのトーンで書かれている
- [ ] 役所の啓発パンフレットのようなレイアウト

**優先度**: P1（重要）

## 非機能要件

### デザイン（最重要）
- フォントは MS Pゴシック 相当（`"MS PGothic", "MS Pゴシック", sans-serif`）を使用する。モダンサンセリフ（Noto Sans等）は使用しない
- 配色は紺（`#003399`）・灰（`#666666`）・くすんだ青（`#336699`）を基調とする
- ヘッダーには「嘘資源管理庁（旧 嘘量調整本部）」を表示し、ローマ字表記なし
- フッターに「Copyright © 平成19年 Ministry of Lie Resources. All rights reserved.」を表示する
- 「本サイトは Internet Explorer 6.0 以上での閲覧を推奨します。」の文言を表示する
- 「最終更新：平成31年4月」を表示する（令和以降の更新なし）
- 左サイドメニューに押せないリンクが並んでいる（ダミーナビゲーション）
- 画面幅は 800px 固定レイアウト（2003年当時の標準）

### パフォーマンス
- プレゼン中にフリーズ・エラーが起きない（デモ安定性最優先）
- Webカメラ起動は 3 秒以内に完了する
- 「解析中...」アニメーションは 3〜5 秒で自動的に結果画面に遷移する

### ユーザビリティ（プレゼンター向け）
- プレゼンターが 1 クリック/タップで各画面に移動できる
- プロジェクターで投影した際、1024×768 以上の解像度で視認できる文字サイズ

### 信頼性
- インターネット接続なしでも動作する（静的ファイルのみで完結）
- 外部APIへの依存なし（グラフデータはすべてハードコード）

### セキュリティ
- Webカメラ映像はブラウザのローカルで完結し、外部送信しない
- 個人情報の収集・保存を一切行わない

---

## スコープ外

- 実際の嘘検出・顔認識・感情分析（すべて演出のみ）
- ユーザーアカウント・ログイン機能
- バックエンドサーバー・データベース
- スマートフォン対応（デスクトップ専用）
- 多言語対応（日本語のみ）
- アクセシビリティ対応（意図的な時代考証優先）
